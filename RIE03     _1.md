## 2. Standard Troubleshooting Procedures (SOP)

### [[Facility Issues]]
- Coordinate with facilities to resolve external utilities (power, gas, cooling) and verify tool recovery.  
  Core action: [Facilities Repair]

---

### [[slit valve issues]]
- Reboot system/RTM and rerun dummy or test wafers to confirm slit valve operation.  
  Core action: [System Reboot]
- Manually pump down TM/LM, then open slit valve and verify transfer.  
  Core action: [Manual Pumpdown]
- Retry slit valve open/transfer sequence and monitor subsequent wafers.  
  Core action: [Transfer Retry]
- Run multiple wafers to attempt to duplicate intermittent slit valve alarms.  
  Core action: [Alarm Reproduce]

---

### [[System Operating issues]]
- Reboot tool/PM computers, reinitialize applications, and verify UI behavior (JOB ID, mapping, etc.).  
  Core action: [Software Reboot]
- Replace or reconnect peripherals (keyboard, mapping sensors, light source, prism mirror, transmitter) and validate cassette mapping.  
  Core action: [IO Hardware Fix]
- Home elevator/robot, remap cassette, and run load/unload tests.  
  Core action: [Robot Rehome]
- Recover pumps/chillers and reboot host after power events.  
  Core action: [Utility Recover]

---

### [[Software, PCS, ELOG issues]]
- Reboot PCS/ELOG host and perform test runs to confirm logging and control stability.  
  Core action: [PCS Reboot]
- Execute D0/qualification flow and prepare SLIM/monitor wafers before releasing to production.  
  Core action: [Qual Flow Setup]

---

### [[Power glitch issues]]
- Recover pumps/chillers, pump chamber down, and safely recover wafers; run dummy wafers to confirm no residual issues.  
  Core action: [Power Recover]

---

### [[Chuck seal issues]]
- Abort alarmed wafers, unload or recover IP wafers, and run dummy wafers to verify He seal stability.  
  Core action: [Seal Check]
- Fail He seal intentionally on dummy, remove dummy, then run production wafer and qualify if needed.  
  Core action: [Dummy Challenge]
- Review prior MESA comments, complete dummy test runs, and then finish queued wafers if no alarms.  
  Core action: [History Review]
- Replace clamp or related hardware when scratches or seal damage are suspected; run dummies and get MEVC approval.  
  Core action: [Clamp Replace]

---

### [[Backside issues]]
- Stop process, unload wafer, and retry with clean/different dummy wafers.  
  Core action: [Dummy Swap]
- Adjust robot Z-height to eliminate backside dragging; verify transfers and mark tool ready for qual.  
  Core action: [Robot Z Adjust]

---

### [[Material or wafer was not detected]]
- Verify wafer location (arm, TM, cassette), correct mapping, and retry transfer.  
  Core action: [Mapping Verify]
- Unload wafers back to cassette and instruct operator to replace specific slots/dummies.  
  Core action: [Wafer Replace]
- Request new Altic dummy wafers instead of worn dummies to improve detection reliability.  
  Core action: [Dummy Refresh]

---

### [[helium took too long to stabilize]]
- Recover wafer IP, run dummy wafers while monitoring He stabilization, and confirm no issues.  
  Core action: [He Stability Check]
- Vent PM to retrieve wafer, run dummies, pump down, and note need for requalification.  
  Core action: [PM Vent]
- Run dummy wafers on He-sensitive recipes (e.g., TMR-FLTR-RIE) to confirm stability.  
  Core action: [He Dummy Test]

---

### [[Others]]
- Adjust sensors (e.g., transfer sensors) and continue RTP after confirming proper operation.  
  Core action: [Sensor Adjust]
- Replace mechanical components (lift bellows, linear slides), perform burn-in and functional tests, then require full quals.  
  Core action: [Mechanical Overhaul]
- Hold tool down for EE to investigate RF1 or TVP high issues; no production until cleared.  
  Core action: [EE Hold]

---

### [[Pressure Out of compliance]]
- Reboot system and re-enable ion gauge; verify gauge operation and pressure readings.  
  Core action: [Gauge Reset]
- Check slots and wafer status when pressure alarms occur; hold affected wafers for engineering review.  
  Core action: [Pressure Audit]

---

### [[Etch Rate issues]]
- Run chamber conditioning with Altic/PR wafers and mark tool ready for qual.  
  Core action: [Chamber Condition]
- Perform wet cleans of chamber, spool, ICP head, clamp/feature plates; then recondition and requal.  
  Core action: [Wet Clean]
- Adjust RF1/RF2 phase/mag, ICP/RF go-on delays (e.g., 5000 ms → 2000 ms), and verify etch rate passes.  
  Core action: [RF Timing Tune]
- Replace critical MFCs (O2, N2), pump out gas lines, recover chamber, and run etch rate quals with RF1 DCV/TV positions checked.  
  Core action: [MFC Replace]
- Modify recipe presets (e.g., step 4 L/T 86/22) or ICP delay per EE, then requal.  
  Core action: [Recipe Preset Tune]
- Perform full etch rate qual (SLIM ER, N2O2, Slim recipes) and log results/spec compliance.  
  Core action: [ER Qualification]

---

### [[Bias issues on RF or recipes]]
- Reset RF generator, vent/inspect clamp plate, replace clamp, and run test wafers; leave chamber vented for full quals.  
  Core action: [Clamp Service]
- Clean ICP head, clamp plate, feature plate, and RF feed-through components; scrub copper ring/plate; then test multiple recipes.  
  Core action: [RF Path Clean]
- Manually set presets/load-tune, test multiple recipes, and keep tool up only for recipes that pass.  
  Core action: [Preset Optimize]
- Tune RF2 match box to minimize reflected power and verify bias at target setpoints.  
  Core action: [Matchbox Tune]

---

### [[RF voltage is out of compliance]]
- Reset RF units/generators, center or fine-tune load/tune capacitors, and run dummy wafers to confirm compliance.  
  Core action: [RF Reset]
- Perform PRC tuning on RF1 load capacitor and adjust RF1 L/T settings (e.g., 55/65, 85/60) to achieve low reflected power.  
  Core action: [RF1 Tune]
- Adjust RF1 reflected tolerance limits (e.g., 100 W → 30 W) and validate via dummy runs on EtchRate/N2O2 recipes.  
  Core action: [Tolerance Set]
- Recover wafers in-process, abort alarmed runs, and rerun dummies/monitor wafers on safe recipes (PMR-MIX2, MPL1024, etc.).  
  Core action: [IP Recover]
- Use recipe gating: restrict tool to known-good recipes; mark N2O2/TMR families down when alarms persist.  
  Core action: [Recipe Gate]
- Investigate slot-dependent RF/DC issues, escalate to EE, and monitor TWS/TIM for alarms.  
  Core action: [Slot Diagnostics]
- Secure RF cables/connectors (e.g., RF2 to ICP head) and retune; declare tool ready for quals.  
  Core action: [RF Cable Fix]

---

### [[Recipe issues]]
- Mark specific recipes or families (MPL1024, N2O2_xxx, TMR-FLTR-RIE, PM3_N202_SCCM) as “do not run” until EE review.  
  Core action: [Recipe Block]
- Run only qualified recipes (MPL1024, PMR-MIX2, PP3_ASH_6W_7M, UU9Q) while problematic recipes are under investigation.  
  Core action: [Qualified-Only]
- Request service actions (S/A) for specific recipes and coordinate EE tuning.  
  Core action: [EE Tuning]
- Return tool to production for non-impacted recipes once prior status and restrictions are reviewed.  
  Core action: [Prod Release]

---

### [[RF generator issues]]
- Place RF generator on dummy load at full power for extended time to verify health; if good, analyze load/tune movements and reflected power.  
  Core action: [Generator Stress Test]
- Replace RF generator and run multi-wafer tests to confirm stability before full quals.  
  Core action: [Generator Replace]
- Coordinate with vendors (Seren, Plasma Therm) for expedited repair and support.  
  Core action: [Vendor Support]

---

### [[RF match box or connection issues]]
- Replace RF1/RF2 match boxes and RF cables; verify new load/tune settings on key recipes (UU6, N2O2, Slim).  
  Core action: [Matchbox Replace]
- Remove RF2 match for external evaluation; reinstall new match box per EE and requal.  
  Core action: [Matchbox Eval]
- Perform chamber surface cleaning and EE tuning of match box before N2O2 qual.  
  Core action: [Matchbox Clean]

---

### [[Tuning issues]]
- Abort wafer, test failing recipes (e.g., N2O2_100s) on dummies, and qualify as needed.  
  Core action: [Dummy Tune Test]
- Work actively on tune-in issues (RF1/RF2 matching) with EE support.  
  Core action: [Tune-In Work]

---

### [[Pump issues]]
- Manually pump down TM/LM and LL; verify stable pressure before resuming production.  
  Core action: [Manual Pump]
- Clean LL door seals and o-rings with appropriate solvents (acetone, IPA), then retry pumpdown.  
  Core action: [Seal Clean]
- Qualify as needed after pump recovery.  
  Core action: [Pump Qual]

---

### [[Scrath issues]]
- Vent tool, replace clamp or scratch-inducing hardware, run multiple dummies, and obtain MEVC approval before production.  
  Core action: [Scratch Clamp Replace]
- Hold tool down for MEVC to inspect scratch issues.  
  Core action: [MEVC Inspect]

---

### [[Qualification Failures]]
- Block specific quals/recipes (e.g., UZ1W on #39887) until MEVC releases; investigate failure data.  
  Core action: [Qual Block]
- Request engineering review when quals fail or alarms persist.  
  Core action: [Eng Review]

---

### [[MFC issues]]
- Replace suspect MFCs (O2, N2), run extended gas flow to condition new MFC/lines, then run dummies to validate.  
  Core action: [MFC Swap]
- Identify correct MFC part numbers and plan replacement schedule.  
  Core action: [Parts Identify]
- After MFC replacement, run multiple dummies and mark tool needing qualification.  
  Core action: [Post-Swap Qual]
